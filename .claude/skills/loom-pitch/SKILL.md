# Loom Pitch

## Структура (максимум 7 хвилин)
- **0:00–0:40 — Проблема.** Хто ти (одним реченням: продуктовий дизайнер, який шипить
  продукти з AI), який біль вирішує продукт. Без історії життя.
- **0:40–4:00 — Демо ядра.** Живий прод-URL, видно адресний рядок. Happy path наскрізь:
  вивалюю реалістичний хаос («подзвонити лікарю, дедлайн по макету в п'ятницю,
  купити корм коту...») → AI структурує → план на сьогодні. Не «task 1, task 2».
- **4:00–5:30 — Рішення.** 2–3 ключові продуктові рішення і чому саме так:
  схема даних як контракт UI↔AI, що свідомо викинуто зі скоупу, mobile-first.
  Говорити як продакт, що приймав рішення.
- **5:30–6:30 — Як допоміг AI.** Конкретика воркфлоу: роадмеп до коду, режими,
  скіли, ітерації по скріншотах. Два живі приклади, не загальні слова.
- **6:30–7:00 — Що далі.** 1–2 advanced-ідеї, які реалізував би наступними, і чому саме їх.

## Правила
- Демо ПЕРЕД поясненнями: спершу показати цінність, потім архітектуру.
- Показуй прод, ніколи не локалку.
- Якщо шматок демо може глюкнути — запиши його окремим дублем заздалегідь.
- Екран + камера одночасно (рекомендація організаторів).
- Тон: впевнено і по суті. Не вибачатися за те, чого не встиг.

## Формат відповіді
Готовий сценарій з репліками по блоках з таймкодами.
Наприкінці — чеклист перед записом: доступ до лінка відкритий «anyone with the link»,
мікрофон перевірений, зайві вкладки й нотифікації закриті, тестовий прогін демо пройдений,
телефонна версія під рукою, якщо демо з телефона.
