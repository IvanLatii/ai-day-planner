# AI Day Planner — фінальне завдання SKELAR AI Lab

## Ролі
Я — продакт-менеджер і продуктовий дизайнер. Ти — розробник, який ніколи не втомлюється.
Я приймаю рішення: що будуємо, що викидаємо, що «достатньо добре». Ти пропонуєш варіанти і виконуєш.
Перед великими задачами: постав уточнюючі питання, покажи план, дочекайся мого ОК. Не вгадуй.
Якщо я кажу «виріши сам» — обирай найпростіший робочий варіант і рухайся далі.

## Продукт
AI-планер дня в стилі Todoist. Користувач вивалює все, що в голові, текстом —
AI перетворює хаос на структуровані задачі та план на сьогодні.
Ядро (має працювати на 100%): Capture → Inbox (AI-парсинг: пріоритет, час, дедлайн) → Today.
Поки ядро не працює наскрізь на проді — жодних інших фіч. Три недороблені функції = нуль продукту.
Голос — апгрейд поверх робочого тексту, не фундамент.

## Стек
- Next.js (App Router) + TypeScript + Tailwind
- Деплой: Vercel, зв'язаний з GitHub. Кожен push = автодеплой. Деплой — перший крок, не останній
- AI: Anthropic API, модель claude-haiku-4-5 для парсингу задач
- API-ключ ТІЛЬКИ в env-змінній ANTHROPIC_API_KEY на сервері (виклик через API route).
  Ніколи в клієнтський код, ніколи в git
- Збереження: почни з найпростішого (localStorage). База даних — тільки якщо ядро вже працює
  і фіча з advanced-списку цього вимагає

## Схема даних — контракт між UI та AI
(чернетка; фіналізується у Фазі 1 після брейншторму)

task: {
  id: string
  title: string
  priority: "low" | "medium" | "high"
  due_date?: string          // ISO date
  time_estimate_min?: number
  tags: string[]
  status: "inbox" | "today" | "done"
  source_text: string        // оригінальний фрагмент вводу
  created_at: string
}

AI повертає СТРОГО JSON-масив таких об'єктів, без жодного тексту навколо.
Відповідь моделі завжди валідується і чиститься: зайвий текст, кривий формат, таймаут —
жодне з цього не має класти UI білим екраном.

## Принципи роботи
- Mobile-first: увесь сценарій має проходитись однією рукою з телефона. Тап-таргети ≥ 44px
- Спочатку «працює», потім «красиво». Але фінальний UI — на рівні: дизайн тут моя перевага
- Empty state нового користувача обов'язковий — це перше, що побачить журі. Порожній екран = виглядає як баг
- Робоча версія завжди закомічена перед експериментами
- Прогін тестів — тільки на прод-URL, з телефона. Локалка ≠ прод (env-змінні!)
- Один сценарій наскрізь важливіший за будь-яку нову фічу
